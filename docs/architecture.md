# OSONE Architecture

## Overview

OSONE is structured as a self-modifying, AI-first operating system. The AI core (skyd) is not a userspace process — it is woven into the kernel itself.

## Layers

```
┌─────────────────────────────────────┐
│         Internet / Research         │  ← skyd reaches out to learn
├─────────────────────────────────────┤
│         I/O & Storage Protocols     │  ← written by skyd
├─────────────────────────────────────┤
│         Driver Layer                │  ← generated by skyd via hardware discovery
├─────────────────────────────────────┤
│         Kernel (self-modifying)     │  ← skyd can rewrite this
├─────────────────────────────────────┤
│         skyd — AI Core (Ring 0)     │  ← Mistral 7B quantized, always running
├─────────────────────────────────────┤
│         Bootloader                  │  ← loads skyd before kernel init
├─────────────────────────────────────┤
│         Hardware                    │
└─────────────────────────────────────┘
```

## Key Principles

- **AI First**: skyd initializes before the kernel is fully assembled
- **Self-Modification**: every layer above the bootloader is writable by skyd
- **Fail Forward**: errors trigger a log → rollback → rewrite cycle
- **Minimal Footprint**: targeting 16–32GB RAM, quantized model inference
- **Emergent Design**: skyd is expected to develop novel solutions over time
